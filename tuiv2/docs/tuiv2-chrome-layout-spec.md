# termx tuiv2 Chrome Layout Spec

状态：Draft
日期：2026-04-03

## 1. 目的

为 workbench / pane / panel chrome 定义稳定的信息分区，避免以下问题继续扩散：

- 文案长短变化导致点击热区漂移
- 同一类状态有时用文本、有时用图标、语义不稳定
- 顶角与底角被临时信息占用，主内容边界不断变化

这里的核心原则不是“把信息塞满”，而是“先固定槽位，再决定内容”。

## 2. 总体原则

- 可点击 chrome 不得由文案自然宽度决定位置。
- 任何会在两种以上状态间切换的按钮，都必须使用固定槽位宽度。
- 长文本只允许出现在标题区和主内容区，不允许占用角落交互位。
- 角落优先放状态、计数、图标、短动作词，不放描述性句子。
- 底边角落优先保留给几何/滚动/溢出提示，不放业务文案。
- workbench 级 chrome 负责全局上下文，pane 级 chrome 只负责局部状态。

## 3. Workbench Chrome

### 3.1 顶栏左侧

固定承载：

- workspace 名称
- tab strip

规则：

- workspace 可以是文本，因为它是定位信息，不是点击热区。
- tab 名称允许截断，但 tab 序号必须稳定保留。
- 顶栏左侧是唯一允许承载中等长度文本的全局 chrome 区域。

### 3.2 顶栏右侧

固定承载：

- notice
- error

规则：

- 这里只允许短暂、临时、全局反馈。
- 不放持续性的运行态摘要，不放模式说明，不放对象名。
- notice / error 消失后，右上角恢复为空，不用其他内容补位。

### 3.3 底栏左侧

固定承载：

- 当前 mode
- 当前 mode 的快捷键提示

规则：

- 用 chip / token 语法展示短文本，不直接输出一整句帮助文案。
- 这里可以密一些，但必须是模式相关，不混入 terminal 状态。
- 默认只显示当前状态下可用的动作。
- 当前状态下不可用的动作优先直接隐藏，不做常驻占位。

### 3.4 底栏右侧

固定承载：

- workspace 摘要
- terminal 数量等全局短摘要

规则：

- 只放全局摘要，不放当前 pane 的 owner / follower / exited。
- 采用短标签语法，例如 `ws:main`、`terms:3`。

## 4. Pane Chrome

### 4.1 顶栏左侧

固定承载：

- terminal title / pane title

规则：

- 这是 pane chrome 唯一允许展示可变长度文本的区域。
- 宽度不足时，优先截断标题，不把标题挤到右侧状态区里。

### 4.2 顶栏右侧

固定槽位，从右向左排列：

1. 角色动作槽位
2. 共享计数槽位
3. 生命周期槽位

规则：

- 角色动作槽位是固定宽度。
- `follow`、`become owner`、`owner` 都必须落在同一宽度槽位内。
- 共享计数使用短文本，例如 `x2`，不展开成句子。
- 生命周期优先用图标或极短 token，例如 `●`、`○42`、`…`、`✕`。
- 所有固定宽度槽位内部文本默认居中。
- 宽度不足时，先丢生命周期，再丢共享计数，最后才丢角色动作槽位。
- 再不足时，整块右侧状态全部让位给标题。

### 4.3 底栏左下 / 右下

固定承载：

- 几何能力提示
- 滚动/溢出提示
- resize affordance

规则：

- 不放 owner / follower / workspace / terminal 名称。
- 不放需要点击的长文本。
- 底角提示必须尽量图形化、方向化、短 token 化。

## 5. Panel / Surface Chrome

适用于 Terminal Pool 等整页 surface。

- 页面标题放左上，不放在右上。
- 页面级筛选、搜索、模式提示放页面头部，不挤占 workbench 顶栏右侧。
- 页面级 footer 只负责当前 surface 的动作，不与全局 status bar 叠加竞争。

## 6. 图标与文本分工

优先用图标：

- 生命周期
- 溢出方向
- 共享/绑定数量的辅助提示

优先用文本：

- workspace 名称
- tab 名称
- pane 标题
- 角色动作词，例如 `follow`、`owner`

禁止使用长文本的区域：

- pane 四角
- workbench 顶栏右侧
- pane 底边两角

## 7. 本轮落地约束

2026-04-03 这一轮先落实以下规则：

- pane 顶栏右侧改为稳定槽位
- owner / follower 相关鼠标点击区固定，不再随文案变化漂移
- 共享关系展示从调试式字符串收敛为短标签

以下仍保留为后续工作：

- workbench 顶栏与底栏的槽位常量化
- pane 底角的正式几何提示体系
- Terminal Pool 页面三栏 chrome 的最终定稿
